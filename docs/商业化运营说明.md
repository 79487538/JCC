# 商业化运营说明

## 卡密售卖流程

推荐流程：

1. 管理员批量生成卡密。
2. 设置卡密有效期，例如 7 天、30 天、90 天。
3. 用户购买后获得卡密。
4. 用户在客户端登录账号。
5. 用户在“卡密”页面输入卡密并激活。
6. 后端记录：
   - 卡密状态
   - 激活用户
   - 激活时间
   - 到期时间
   - 设备限制

建议后续增加：

- 卡密批量生成后台。
- 卡密订单关联。
- 卡密使用记录。
- 到期提醒。
- 续费入口。

## 普通用户和高级用户区别

可按权限设计用户等级。

普通用户：

- 使用规则推荐。
- 使用低成本 AI 模型。
- 每日有限次数。
- 不支持高级截图识别模板。

高级用户：

- 使用更高性能模型。
- 更高每日请求次数。
- 支持截图识别区域模板保存。
- 支持优先更新版本。
- 支持更多阵容策略配置。

建议通过卡密字段或用户权限表扩展：

- plan
- quota_daily
- model_level
- expires_at
- device_limit

## AI 模型成本说明

AI 调用成本由以下因素决定：

- 输入 token 数量。
- 输出 token 数量。
- 模型单价。
- 用户请求频率。

当前后端会返回：

- `model_used`
- `estimated_cost_usd`
- `ai_status`
- `ai_error`

运营建议：

- 默认用户优先使用低成本模型。
- 关键回合或低血量场景才使用高性能模型。
- 对用户设置每日 AI 请求额度。
- 记录每次调用的估算成本。
- 定期统计用户维度、模型维度、日期维度成本。

建议模型策略：

- 普通场景：DeepSeek Flash、Qwen Plus、GPT-4o-mini。
- 高压场景：Qwen Max、GPT-4o、DeepSeek R1。
- 备用聚合：APIRouter / AIPower。

## 用户反馈收集建议

建议收集以下反馈：

- 推荐是否有帮助。
- OCR 识别是否准确。
- 哪些字段经常识别失败。
- 哪些阵容建议不合理。
- 是否希望增加自动阵容模板。
- 是否希望增加历史对局记录。

可在客户端增加：

- 一键反馈按钮。
- 推荐评分。
- OCR 错误上报。
- 用户备注。

反馈数据建议包含：

- 用户 ID。
- 软件版本。
- 后端版本。
- 局势 JSON。
- 推荐结果。
- 用户评分。
- 用户备注。

## 版本更新建议

建议采用小步快跑：

1. 先稳定登录、卡密、手动推荐。
2. 再优化截图识别准确率。
3. 再增加阵容版本库。
4. 再增加用户套餐和额度。
5. 最后增加运营后台。

版本发布建议：

- 每次发布记录 changelog。
- 保留旧版本安装包。
- 新版本先小范围灰度。
- 收集崩溃、接口失败、OCR 失败反馈。
- 对模型成本设置报警阈值。

商业化初期应优先控制：

- AI 成本。
- OCR 准确率。
- 用户登录和卡密稳定性。
- 安装包可用性。
